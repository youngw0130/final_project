## 🚀 1. 실행

터미널 한 개에서 다음 한 줄만 실행하면 됩니다.

```bash
bash scripts/start-all.sh
```

- 백엔드(Spring Boot) → 8080 포트
- 프론트(정적 HTML) → 3000 포트
- 둘 다 정상 부팅되면 콘솔에 모든 접속 URL이 출력됩니다.

부팅에 1~2분 소요됩니다. (처음 한 번만 Gradle 다운로드)

## 🎯 2. 핵심 데모 시나리오 (3분 버전)

### Step 1. 프로토타입 허브 열기
👉 [http://localhost:3000](http://localhost:3000)

> "여기가 크레딧-N 프로토타입 허브입니다. 6개의 화면 디자인과 백엔드와 연동된 데모 패널이 있습니다."

### Step 2. 데모 컨트롤 패널 진입
👉 [http://localhost:3000/demo.html](http://localhost:3000/demo.html)

> "오른쪽에 모든 API 호출 로그가 실시간으로 표시됩니다."

### Step 3. 로그인
- 아이디 `감나빗`, 비밀번호 `test1234` 입력
- **[로그인]** 버튼 클릭

> "JWT 토큰을 발급받고, 우상단에 사용자 정보가 표시됩니다. 링크 스코어도 보이죠."

### Step 4. 내 모임 조회
- **[📋 내 모임 조회]** 클릭

> "지금 사용자가 참여 중인 모임 2개가 보입니다. 모임 1은 ACTIVE, 모임 2는 OPEN 상태예요. 각 모임마다 자동 발급된 가상계좌 번호가 있습니다."

### Step 5. 모임 1 상세 조회
- 모임 ID `1`인 상태로 **[상세] → [참여자] → [결제 내역]** 차례 클릭

> "참여자 4명 모두 DEPOSITED 상태이고, QR 결제는 3건이 누적되어 있습니다. 잔액이 15,500원이네요."

### Step 6. QR 결제 시연
- 가맹점명, 카테고리, 금액 자동 채워져 있음 (`라멘야 부산본점 / 식음료 / 15000`)
- **[📱 QR 결제 승인]** 클릭

> "결제 즉시 잔액에서 차감됩니다. 한 번 더 [내 모임 조회] 해보면 totalSpent가 증가한 게 보입니다."

### Step 7. 정산 실행
- **[📊 정산 실행]** 클릭

> "넷팅 정산으로 총지출을 4명에게 1/4씩 분배합니다. 환급액이 자동 계산되고 상태가 REFUNDED, 모임은 CLOSED로 전환됩니다."

### Step 8. (보너스) OPEN → ACTIVE 자동 전환 시연
- 모임 ID를 `2`로 변경
- userId `2`(minsu) 입력 → **[💰 입금 확정]**
- userId `4`(hyunwoo) 입력 → **[💰 입금 확정]**
- **[상세]** 클릭

> "전원 입금이 완료되면 모임 상태가 OPEN → ACTIVE로 자동 전환됩니다."

## 🎨 3. 디자인 화면 시연

데모 패널 시연 후 디자인 시안을 보여줄 때:

1. 우상단 **[← 허브]** 버튼 클릭
2. 보여주고 싶은 화면 클릭 (특히 추천: **01 대시보드**, **03 에스크로 상세**)
3. 화면 우상단에 떠 있는 **[▶ 데모 패널]** 버튼으로 다시 패널로 복귀

### 대시보드 화면 팁
로그인된 상태로 [01 대시보드](http://localhost:3000/prototypes/01-dashboard.html)를 열면, **상단 인사말의 사용자 이름과 링크 스코어가 실제 백엔드 데이터로 자동 치환**됩니다.

> "디자인 화면도 백엔드와 연결되어 있어, 로그인한 사용자 정보가 그대로 반영됩니다."

## 🔧 4. 트러블슈팅

### 백엔드가 안 켜져요
```bash
cat .run/backend.log | tail -50
```
대부분 포트 8080이 사용 중인 경우입니다. `lsof -i :8080`으로 확인하고 종료 후 재시작.

### 시드 데이터가 사라졌어요
H2 인메모리 DB라서 서버 재시작 시 매번 새로 생성됩니다. 정상입니다.

### JWT 토큰이 만료됐어요
24시간이 기본 만료입니다. 데모 패널에서 다시 [로그인] 누르면 됩니다.

### CORS 에러가 떠요
`http://127.0.0.1:3000` 대신 `http://localhost:3000`으로 접속하세요. SecurityConfig에서 `*` origin 허용 중이니 일반적으로는 문제 없습니다.

## 📊 5. 백엔드 API 요약 (구현 완료)

### 인증
| 메서드 | 엔드포인트 | 설명 |
|---|---|---|
| POST | `/api/auth/signup` | 회원가입 + JWT 발급 |
| POST | `/api/auth/login` | 로그인 + JWT 발급 |

### 모임
| 메서드 | 엔드포인트 | 설명 |
|---|---|---|
| POST | `/api/moims` | 모임 생성 + 가상계좌 자동 발급 (mock) |
| POST | `/api/moims/join?inviteCode=...` | 초대코드로 참여 |
| GET  | `/api/moims/{id}` | 모임 상세 |
| GET  | `/api/moims/my` | 내가 참여 중인 모임 목록 |
| GET  | `/api/moims/{id}/participants` | 참여자 목록 |
| POST | `/api/moims/{id}/deposit/confirm?userId=` | 입금 확정 (PortOne 웹훅 mock) |
| POST | `/api/moims/{id}/settle` | 넷팅 정산 + 환급 처리 |

### 결제 (QR)
| 메서드 | 엔드포인트 | 설명 |
|---|---|---|
| POST | `/api/moims/{id}/payments` | QR 결제 승인 (mock) |
| GET  | `/api/moims/{id}/payments` | 결제 내역 |

### 상태 자동 전환
- 모임 생성 → `OPEN` + 가상계좌 즉시 발급
- 전원 입금 완료 → `OPEN` → **`ACTIVE`** (QR 결제 활성화)
- 정산 호출 → `ACTIVE` → `SETTLING` → `CLOSED` (환급까지 자동)
- 입금 기한 초과 → `PENDING` → `OVERDUE` (1시간 스케줄러)

> 핵심은 두 가지인데요. 첫째, 누구 한 사람이 돈을 들고 도망갈 걱정이 없습니다.

> 둘째, 모임 끝나면 남은 잔액을 1/n로 자동 분배합니다. 더치페이 정산 다툼 끝.
>
> 추가로, 모든 입금/결제 기록이 '링크 스코어'라는 대안 신용 점수로 누적됩니다.
> 기존 신용 점수 시스템이 닿지 못하는 학생이나 사회 초년생도 자신의 신뢰도를 데이터로 입증할 수 있어요.
